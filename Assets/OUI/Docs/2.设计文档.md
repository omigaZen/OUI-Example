## 一、整体架构

```
OUI (管理器, MonoBehaviour单例)
 ├── UIRoot (Transform, 所有Window挂载于此)
 ├── IWindowAssetLoader (资源加载接口, 外部注入)
 └── _stack: List<BaseWindow> (有序窗口栈)
       ├── [0] WindowA (Bottom层)
       ├── [1] WindowB (UI层, 先打开)
       ├── [2] WindowC (UI层, 后打开, sortingOrder更大)
       └── [3] WindowD (System层)

BaseWindow (MonoBehaviour, 挂在Prefab根节点)
 ├── Canvas / GraphicRaycaster 引用
 ├── Depth (sortingOrder)
 ├── _hideByAbove (被上层遮挡)
 ├── _closed (被关闭)
 ├── 隐藏 = _hideByAbove || _closed (统一Layer切换)
 └── 生命周期方法
```

## 二、窗口栈实现

### 2.1 数据结构

使用 `List<BaseWindow>` 作为有序栈（非传统LIFO栈），始终保持按Layer排序，同Layer内按打开顺序排列。

### 2.2 Push（插入）

```C#
private void Push(BaseWindow window)
{
    // 找到同Layer的最后一个窗口，插入其后面
    int insertIndex = -1;
    for (int i = 0; i < _stack.Count; i++)
    {
        if (window.WindowLayer == _stack[i].WindowLayer)
            insertIndex = i + 1;
    }

    // 如果该Layer没有窗口，找相邻的更低Layer的最后一个窗口，插入其后面
    if (insertIndex == -1)
    {
        for (int i = 0; i < _stack.Count; i++)
        {
            if (window.WindowLayer > _stack[i].WindowLayer)
                insertIndex = i + 1;
        }
    }

    // 栈为空或该Layer最低，插入到索引0
    if (insertIndex == -1)
        insertIndex = 0;

    _stack.Insert(insertIndex, window);
}
```

### 2.3 Pop（移除）

```C#
private void Pop(BaseWindow window)
{
    _stack.Remove(window);
}
```

## 三、层级与排序实现

### 3.1 排序常量

```C#
public const int LAYER_DEEP = 2000;   // 层级之间的sortingOrder间隔
public const int WINDOW_DEEP = 100;   // 同层级内窗口之间的sortingOrder间隔
```

各层起始sortingOrder：Bottom=0, UI=2000, Top=4000, Tips=6000, System=8000

### 3.2 sortingOrder计算

```C#
private void OnSortWindowDepth(WindowLayer layer)
{
    int depth = (int)layer * LAYER_DEEP;
    for (int i = 0; i < _stack.Count; i++)
    {
        if (_stack[i].WindowLayer == layer)
        {
            _stack[i].Depth = depth;
            depth += WINDOW_DEEP;
        }
    }
}
```

### 3.3 Depth属性实现

设置 `Canvas.sortingOrder`，子Canvas基于父值递增+5。

> **约束**：每个窗口的子Canvas数量不应超过 `WINDOW_DEEP / 5 = 20` 个，否则sortingOrder会溢出到下一个窗口的范围。实现时需加断言检查。

```C#
public int Depth
{
    set
    {
        Debug.Assert(_childCanvas.Length <= WINDOW_DEEP / 5,
            $"Window子Canvas数量({_childCanvas.Length})超过上限({WINDOW_DEEP / 5})，sortingOrder将溢出");

        _canvas.sortingOrder = value;
        int childDepth = value;
        for (int i = 0; i < _childCanvas.Length; i++)
        {
            if (_childCanvas[i] != _canvas)
            {
                childDepth += 5;
                _childCanvas[i].sortingOrder = childDepth;
            }
        }
    }
}
```

## 四、窗口可见性实现

### 4.1 隐藏方式：Layer切换（而非SetActive）

Window的关闭和HideBelow遮挡都使用同一种隐藏机制——切换Unity Layer + 禁用GraphicRaycaster：

> **前提条件**：UI Camera 的 culling mask 必须包含 `WINDOW_SHOW_LAYER`（UI层=5），且不包含 `WINDOW_HIDE_LAYER`（Ignore Raycast层=2）。实现代码中需在 BaseWindow 类上方注释区域提示用户此要求。

| | `SetActive(false)` | **Layer切换 + 禁用Raycaster** |
|---|---|---|
| Canvas Rebuild | 重新激活时触发重建 | 不触发，Canvas始终存活 |
| 协程/Update | 全部停止 | 继续运行 |
| 动画/粒子 | 停止，恢复后状态可能不对 | 继续播放 |
| OnEnable/OnDisable | 递归触发 | 不触发 |

### 4.2 双布尔值可见性模型

Window维护两个独立的隐藏原因，任意一个为true则隐藏：

```C#
private bool _hideByAbove = false;  // 被上层HideBelow窗口遮挡
private bool _closed = false;       // 被关闭（初始值为false，首次创建时窗口默认为打开状态）

// 是否需要隐藏
private bool IsHidden => _hideByAbove || _closed;
```

### 4.3 统一的可见性刷新

两个bool任意变化时，统一调用同一个方法刷新显示状态：

```C#
private Canvas _canvas;
private Canvas[] _childCanvas;
private GraphicRaycaster _raycaster;
private GraphicRaycaster[] _childRaycaster;

private void RefreshVisible()
{
    bool visible = !IsHidden;
    int setLayer = visible ? WINDOW_SHOW_LAYER : WINDOW_HIDE_LAYER;
    // WINDOW_SHOW_LAYER = 5 (UI层)
    // WINDOW_HIDE_LAYER = 2 (Ignore Raycast层)

    _canvas.gameObject.layer = setLayer;
    foreach (var child in _childCanvas)
        child.gameObject.layer = setLayer;

    _raycaster.enabled = visible;
    foreach (var child in _childRaycaster)
        child.enabled = visible;
}

public bool HideByAbove
{
    get => _hideByAbove;
    set
    {
        if (_hideByAbove == value) return;
        _hideByAbove = value;
        RefreshVisible();
        OnHideByAboveChanged(value);
    }
}
```

### 4.4 HideBelow可见性遍历算法

每次有窗口打开、关闭、销毁时都需要调用。从栈顶向下扫描所有已打开的窗口，找到第一个 `HideBelow=true` 的窗口，该窗口及其上方的已打开窗口可见，该窗口下方的所有已打开窗口不可见：

```C#
private void OnSetWindowVisible()
{
    bool isHideNext = false;
    for (int i = _stack.Count - 1; i >= 0; i--)
    {
        var window = _stack[i];
        if (window.IsClosed) continue;  // 已关闭的窗口不参与遮挡计算

        if (!isHideNext)
        {
            window.HideByAbove = false;
            if (window.HideBelow)
                isHideNext = true;
        }
        else
        {
            window.HideByAbove = true;
        }
    }
}
```

## 五、窗口生命周期流程

### 5.1 首次打开流程（ShowUIAsync，窗口不存在）

```
1. 检查栈中是否已存在该类型Window → 不存在
2. 检查是否正在加载中 → 否，标记为加载中，继续
3. 反射读取 [WindowAttribute] → 获取Location, Layer, HideBelow
4. IWindowAssetLoader.LoadWindowAsync(location) → 异步加载Prefab
5. Instantiate到UIRoot下 → 获取/添加BaseWindow组件
6. 缓存Canvas、Canvas[]、GraphicRaycaster、GraphicRaycaster[]引用
7. Push到栈中
8. OnSortWindowDepth(layer) → 重算同层sortingOrder
9. BindUI() → Init()
10. OnOpen() / OnOpen(args)
11. OnSetWindowVisible() → 重算所有窗口HideBelow可见性
12. 清除加载中标记
```

### 5.2 防重入

- 窗口已打开（存在且 `IsClosed == false`）：直接return
- 窗口资源正在加载中：直接return
- 使用 `HashSet<Type>` 记录正在加载的窗口类型，加载开始时加入，加载完成后移除

### 5.3 重新打开流程（ShowUIAsync，窗口已存在且已关闭）

```
1. 检查栈中已存在该类型Window，且 IsClosed == true
2. Pop → Push（移到同Layer栈顶）
3. OnSortWindowDepth(layer) → 重算同层sortingOrder
4. _closed = false → RefreshVisible()（通过Layer切换显示）
5. OnOpen() / OnOpen(args)
6. OnSetWindowVisible() → 重算所有窗口HideBelow可见性
```

### 5.4 关闭流程（CloseWindow）— 不销毁

```
1. OnClose()
2. _closed = true → RefreshVisible()（通过Layer切换隐藏）
3. OnSetWindowVisible() → 重算所有窗口HideBelow可见性
```

注意：关闭不会从栈中移除，不会销毁GameObject，不会调用Release()。

### 5.5 销毁流程（DestroyWindow）— 真正销毁

```
1. 如果未关闭，先执行关闭流程
2. Release()
3. Pop从栈中移除
4. Destroy(gameObject)
5. OnSortWindowDepth(layer) → 重算同层sortingOrder
6. OnSetWindowVisible() → 重算所有窗口HideBelow可见性
```

### 5.6 生命周期总览

```
加载 → BindUI() → Init() → OnOpen() ⇄ OnClose() → Release() → Destroy
       (仅首次)   (仅首次)    ↕  (可反复Open/Close)   (DestroyWindow时)
                          OnHideByAboveChanged(hidden)
```
