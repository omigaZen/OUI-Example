## 一、整体架构

```
OUI (管理器, MonoBehaviour单例)
 ├── UIRoot (Transform, 所有Window挂载于此)
 ├── IWindowAssetLoader (资源加载接口, 外部注入)
 ├── _stack: List<BaseWindow> (有序窗口栈)
 ├── _loadingWindows: HashSet<Type> (正在加载中的Window类型)
 ├── _pendingShowQueues: Dictionary<Type, Queue<object[]>> (重复打开时的待续开队列)
 └── _suppressQueuedShowDrain: bool (批量关闭/销毁时禁止自动续开)

BaseWindow (MonoBehaviour, 挂在Prefab根节点)
 ├── Canvas / GraphicRaycaster 引用
 ├── Depth (sortingOrder)
 ├── _hideByAbove (被上层遮挡)
 ├── _closed (被关闭)
 ├── 隐藏 = _hideByAbove || _closed (统一Layer切换)
 └── 生命周期方法
```

## 二、窗口栈实现

### 2.1 数据结构

- 使用 `List<BaseWindow>` 作为有序栈（非传统LIFO栈），始终保持按Layer排序，同Layer内按打开顺序排列。
- 使用 `HashSet<Type>` 记录正在加载中的窗口类型，避免首次打开阶段重入。
- 使用 `Dictionary<Type, Queue<object[]>>` 记录开启了 `QueueAfterClose` 的窗口排队请求，队列顺序为 FIFO。

### 2.2 Push（插入）

```C#
private void Push(BaseWindow window)
{
    int insertIndex = -1;
    for (int i = 0; i < _stack.Count; i++)
    {
        if (window.WindowLayer == _stack[i].WindowLayer)
            insertIndex = i + 1;
    }

    if (insertIndex == -1)
    {
        for (int i = 0; i < _stack.Count; i++)
        {
            if (window.WindowLayer > _stack[i].WindowLayer)
                insertIndex = i + 1;
        }
    }

    if (insertIndex == -1)
        insertIndex = 0;

    _stack.Insert(insertIndex, window);
}
```

### 2.3 Pop（移除）

```C#
private void Pop(BaseWindow window)
{
    _stack.Remove(window);
}
```

## 三、层级与排序实现

### 3.1 排序常量

```C#
public const int LAYER_DEEP = 2000;   // 层级之间的sortingOrder间隔
public const int WINDOW_DEEP = 100;   // 同层级内窗口之间的sortingOrder间隔
```

各层起始sortingOrder：Bottom=0, UI=2000, Top=4000, Tips=6000, System=8000

### 3.2 sortingOrder计算

```C#
private void OnSortWindowDepth(WindowLayer layer)
{
    int depth = (int)layer * LAYER_DEEP;
    for (int i = 0; i < _stack.Count; i++)
    {
        if (_stack[i].WindowLayer == layer)
        {
            _stack[i].Depth = depth;
            depth += WINDOW_DEEP;
        }
    }
}
```

### 3.3 Depth属性实现

设置 `Canvas.sortingOrder`，子Canvas基于父值递增+5。

> **约束**：每个窗口的子Canvas数量不应超过 `WINDOW_DEEP / 5 = 20` 个，否则sortingOrder会溢出到下一个窗口的范围。实现时需加断言检查。

```C#
public int Depth
{
    set
    {
        Debug.Assert(_childCanvas.Length <= WINDOW_DEEP / 5,
            $"Window子Canvas数量({_childCanvas.Length})超过上限({WINDOW_DEEP / 5})，sortingOrder将溢出");

        _canvas.sortingOrder = value;
        int childDepth = value;
        for (int i = 0; i < _childCanvas.Length; i++)
        {
            if (_childCanvas[i] != _canvas)
            {
                childDepth += 5;
                _childCanvas[i].sortingOrder = childDepth;
            }
        }
    }
}
```

## 四、窗口可见性实现

### 4.1 隐藏方式：Layer切换（而非SetActive）

Window的关闭和HideBelow遮挡都使用同一种隐藏机制：切换Unity Layer + 禁用GraphicRaycaster。

> **前提条件**：UI Camera 的 culling mask 必须包含 `WINDOW_SHOW_LAYER`（UI层=5），且不包含 `WINDOW_HIDE_LAYER`（Ignore Raycast层=2）。实现代码中需在 `BaseWindow` 类上方注释区域提示用户此要求。

| | `SetActive(false)` | **Layer切换 + 禁用Raycaster** |
|---|---|---|
| Canvas Rebuild | 重新激活时触发重建 | 不触发，Canvas始终存活 |
| 协程/Update | 全部停止 | 继续运行 |
| 动画/粒子 | 停止，恢复后状态可能不对 | 继续播放 |
| OnEnable/OnDisable | 递归触发 | 不触发 |

### 4.2 双布尔值可见性模型

Window维护两个独立的隐藏原因，任意一个为true则隐藏：

```C#
private bool _hideByAbove = false;  // 被上层HideBelow窗口遮挡
private bool _closed = false;       // 被关闭（首次创建时默认为打开状态）

private bool IsHidden => _hideByAbove || _closed;
```

### 4.3 统一的可见性刷新

```C#
private void RefreshVisible()
{
    bool visible = !IsHidden;
    int setLayer = visible ? WINDOW_SHOW_LAYER : WINDOW_HIDE_LAYER;

    _canvas.gameObject.layer = setLayer;
    foreach (var child in _childCanvas)
    {
        if (child != _canvas)
            child.gameObject.layer = setLayer;
    }

    _raycaster.enabled = visible;
    foreach (var child in _childRaycaster)
    {
        if (child != _raycaster)
            child.enabled = visible;
    }
}
```

### 4.4 HideBelow可见性遍历算法

每次有窗口打开、关闭、销毁时都重新计算。只遍历当前处于打开状态的窗口，从栈顶向下找到第一个 `HideBelow=true` 的窗口，其下方所有已打开窗口都会被遮挡：

```C#
private void OnSetWindowVisible()
{
    bool isHideNext = false;
    for (int i = _stack.Count - 1; i >= 0; i--)
    {
        BaseWindow window = _stack[i];
        if (window.IsClosed)
            continue;

        if (!isHideNext)
        {
            window.SetHideByAbove(false);
            if (window.HideBelow)
                isHideNext = true;
        }
        else
        {
            window.SetHideByAbove(true);
        }
    }
}
```

## 五、重复打开策略与生命周期

### 5.1 Attribute配置

`WindowAttribute` 新增 `DuplicateShowMode`：

```C#
public enum WindowDuplicateShowMode
{
    Return = 0,
    QueueAfterClose = 1,
}

[AttributeUsage(AttributeTargets.Class)]
public class WindowAttribute : Attribute
{
    public string AssetPath { get; }
    public WindowLayer Layer { get; }
    public bool HideBelow { get; }
    public WindowDuplicateShowMode DuplicateShowMode { get; set; }
        = WindowDuplicateShowMode.Return;
}
```

说明：
- 默认值仍为 `Return`，确保既有窗口行为不变。
- 仅显式声明 `QueueAfterClose` 的窗口才会进入排队模式。

### 5.2 参数快照规则

待打开队列缓存的是 `object[]` 的浅拷贝快照：

```C#
private static object[] SnapshotArgs(object[] args)
{
    if (args == null || args.Length == 0)
        return Array.Empty<object>();

    object[] snapshot = new object[args.Length];
    Array.Copy(args, snapshot, args.Length);
    return snapshot;
}
```

说明：
- 复制数组本身，避免调用方复用同一个 `object[]` 后修改内容。
- 不深拷贝数组内的引用类型对象，保持原需求中的“传引用”语义。

### 5.3 首次打开流程（窗口不存在）

```
1. 读取 WindowAttribute，拿到 AssetPath / Layer / HideBelow / DuplicateShowMode
2. 若不在加载中，则加入 _loadingWindows
3. 调用 IWindowAssetLoader.LoadAsync(assetPath)
4. Instantiate 到 UIRoot 下
5. 获取或补挂 BaseWindow 组件
6. InternalInit() 缓存 Canvas / Raycaster 引用
7. Push 到 _stack
8. OnSortWindowDepth(layer)
9. BindUI() -> Init()
10. InternalOpen() -> OnOpen(args)
11. OnSetWindowVisible()
12. finally 中移除 _loadingWindows 标记
```

### 5.4 已存在且已关闭：复用实例重新打开

```C#
private void ReopenWindow(BaseWindow window, object[] args)
{
    Pop(window);
    Push(window);
    OnSortWindowDepth(window.WindowLayer);
    window.InternalOpen();
    window.OnOpen(args);
    OnSetWindowVisible();
}
```

说明：
- 仍然保持“单例 + Close=Hide”模型。
- 重新打开时不重新加载资源，不重新执行 `BindUI()` / `Init()`。

### 5.5 已存在且已打开 / 正在加载：重复打开策略

#### 5.5.1 Return 模式

- 若窗口已打开：直接 return
- 若窗口正在加载：直接 return

#### 5.5.2 QueueAfterClose 模式

- 若窗口已打开：把参数快照入队
- 若窗口正在加载：把参数快照入队
- 队列按 `Type -> Queue<object[]>` 维护，顺序为 FIFO

```C#
private void HandleDuplicateShow(Type type, WindowAttribute attr, object[] args)
{
    if (attr.DuplicateShowMode != WindowDuplicateShowMode.QueueAfterClose)
        return;

    EnqueuePendingShow(type, args);
}
```

### 5.6 关闭流程（CloseWindow）

```C#
private void CloseWindowInternal(BaseWindow window, bool triggerQueuedShow)
{
    if (window.IsClosed)
        return;

    window.OnClose();
    window.InternalClose();
    OnSetWindowVisible();

    if (triggerQueuedShow)
        TryShowNextPendingWindow(window.GetType());
}
```

说明：
- 仅当窗口本次确实从打开态变为关闭态时，才允许触发自动续开。
- 自动续开本质上仍然走一次框架内部的 `ShowUIAsync(type, args)`，因此会重新参与层级排序和 `HideBelow` 计算。

### 5.7 自动续开流程

```C#
private void TryShowNextPendingWindow(Type type)
{
    if (_suppressQueuedShowDrain)
        return;

    if (!TryDequeuePendingShow(type, out object[] nextArgs))
        return;

    ContinuePendingShowAsync(type, nextArgs).Forget();
}
```

说明：
- `TryDequeuePendingShow` 取出队首参数，队列空时移除字典项。
- `ContinuePendingShowAsync` 内部继续调用统一打开入口。
- 若自动续开失败，会记录错误日志；当前类型的待打开队列会在失败清理阶段清空。

### 5.8 销毁流程（DestroyWindow）

```
1. ClearPendingShows(type)
2. 进入 _suppressQueuedShowDrain = true，禁止 OnClose 期间触发自动续开
3. 若窗口当前是打开态，则执行 OnClose() -> InternalClose()
4. Release()
5. Pop 从 _stack 移除
6. AssetLoader.Unload(window.gameObject)
7. Destroy(window.gameObject)
8. OnSortWindowDepth(layer)
9. OnSetWindowVisible()
```

说明：
- 销毁优先级高于排队续开，销毁后不会再自动打开旧请求。
- 同类型待打开队列会被立即清空。

### 5.9 批量关闭流程（CloseAllWindows）

```C#
public void CloseAllWindows()
{
    _suppressQueuedShowDrain = true;
    try
    {
        foreach (var window in _stack)
        {
            if (!window.IsClosed)
            {
                window.OnClose();
                window.InternalClose();
            }
        }

        ClearAllPendingShows();
        OnSetWindowVisible();
    }
    finally
    {
        _suppressQueuedShowDrain = false;
    }
}
```

说明：
- 批量关闭过程中，任何 `QueueAfterClose` 窗口都不得自动续开。
- 设计上直接清空全部待打开队列，保证 `CloseAllWindows()` 返回后得到稳定结果：所有窗口关闭，且没有旧请求延后弹出。

### 5.10 全部销毁流程（DestroyAll）

- 先进入 `_suppressQueuedShowDrain = true`
- 清空全部待打开队列
- 倒序遍历 `_stack`，对每个窗口执行 `OnClose()`（如仍打开）、`Release()`、`Unload()`、`Destroy()`
- 最后 `_stack.Clear()`

### 5.11 异常与失败处理

- `WindowAttribute` 缺失：直接抛异常
- `AssetLoader` 未设置：直接抛异常
- 资源加载、实例化、补挂组件、`BindUI()`、`Init()`、`OnOpen()` 任意阶段失败：
  - 回滚已创建但未完成初始化的实例
  - 从 `_stack` 中移除半成品窗口（如果已经入栈）
  - 清空该类型待打开队列
  - 输出错误日志
  - 移除 `_loadingWindows` 标记

### 5.12 生命周期总览

```
首次打开:
加载 -> BindUI() -> Init() -> OnOpen(args)

重复打开:
- Return: 直接return
- QueueAfterClose: 入 _pendingShowQueues

关闭:
OnClose() -> InternalClose() -> OnSetWindowVisible()
                             -> TryShowNextPendingWindow(type)

销毁:
OnClose() -> Release() -> Pop -> Unload/Destroy
```

## 六、兼容性结论

- 未配置 `DuplicateShowMode` 的窗口仍保持旧行为：已打开/加载中时直接 return。
- `BaseWindow` 对外接口不变，业务窗口仍只需在 `OnOpen(args)` 中消费参数。
- 自动续开的窗口仍参与原有窗口栈排序、`HideBelow` 遮挡和生命周期回调，不需要业务层写额外逻辑。
